This stage only filters keys; it assigns no values, builds no persona, and never claims the real wearer has any attribute.

Analysis & honest caveats

  • These are model preferences, not ground truth. Selection frequency measures what the VLM thinks is useful, not whether the key actually improves deviation data. The spec (§18) requires a human-annotated subset with precision/recall/F1/IAA before trusting these rates — not done here.
  • The prompt/threshold were frozen after a 2-video inspection, not tuned on this corpus — so selection rates are a first pass, not calibrated numbers.
  • Task-conditioned, as intended: capability/skill and safety keys dominate on multi-step procedural videos; social and narrative (McAdams Level III) keys are rarely selected because these egocentric task videos give them little deviation leverage. A rarely-selected key can still be crucial for a small subset (§5.4).
  • Sensitive keys 🔒 can be selected as candidate conditioning dimensions, but the reasons were checked to be task-mechanism-based, never appearance inference (§5.3). Spot-check them before use.
  • 97 keys were never selected on any video (see below) — mostly Level-III narrative-identity keys with no concrete action-deviation mechanism in these tasks.
