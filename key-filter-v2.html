Simplified: no reason codes. For each selected key the VLM names a CONCRETE deviation it could cause and the time window where it would happen. Click a time chip to jump the video there. 4 videos.

How to read these — they are illustrative COUNTERFACTUAL examples, not facts about the video.
  • The video shows the correct execution (the reference trajectory). Each line is a hypothetical: "IF the user had this persona (e.g. a novice), here is the deviation they would make, and when." It never claims the person on camera actually did this, and selecting a key never claims the real wearer has that attribute.
  • So a video where the peanut butter is spread evenly is not a contradiction — it is the correct baseline the counterfactual deviation is defined against. Keeping the video correct and synthesizing the deviation is exactly how user-deviation data is built.
  • These lines are a preview + evidence + seed for the next stage (actually generating the deviation samples). No deviation data is generated here. Use the time chips to check that each hypothetical deviation is genuinely possible at that moment.
